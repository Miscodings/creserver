2D Sprites

Thankfully, 2D sprites are actually a little easier (in my experience) than 3D models, mainly because there's one less step involved, and instead, one more config option!

Item Config → Texture

Config (.yml)

→

Texture (.png)

As you can see, there is no longer a model .json file involved, and the item links directly up to a texture in the correct path. The configuration of an item points to the texture that the item should use. However, you now have to let Minecraft know how exactly you want this item to be displayed in your hand.

Here's An Example:

Nexo/items/skill_icons.yml

dagger_throw_icon:
material: PAPER
Pack:
  parent_model: item/generated
  textures:
  - skill_icons/dagger_throw
  custom_model_data: 1000

- `dagger_throw_icon` is the ID of the item. Every item has its own, unique ID.

- The base item, or `material:` that this axe is using is a Paper.

- This item does not have a display name.

- In terms of the pack, it's custom-model-data is being assigned to the number 1,000.

The texture path is being assigned to a .png file located in the directory skill_icons/dagger_throw. That's in:

Nexo/pack/assets/minecraft/textures/skill_icons/dagger_throw.png

Before we move on, let's talk about what `parent_model` means.

Parent Models

To put it simply, the `parent_model` configuration option just determines how your item looks when it's held.

You may have noticed that holding a FEATHER or a PAPER in your hand, looks different from holding a SWORD or a STICK. That's due to the parent_models of those items being different.

For items that should be held horizontally like PAPER or FEATHERs, use 'parent_model: item/generated'.

For items that should be held vertically like SWORDs or TOOLs, use 'parent_model: item/handheld'.

So pretty much, if you're creating 2D tools or weapons, item/handheld is the way to go. Otherwise, for like ores and stuff, you're likely going to want to stick with item/generated.
